1. 产品概述
1.1 系统定位
本平台是「AI音乐教学管理 + 版权代理发行」的一体化运营系统，服务于以AI工具（汽水创作实验室、Suno等）进行音乐创作的教育机构及个人创作者。
关键区分：本平台 ≠ 汽水音乐。汽水音乐是字节旗下独立流媒体平台，拥有自身ID体系和收益数据。本平台维护内部独立编号体系（AIMU-YYYY-NNNNNN），通过「歌曲ID映射表」单向桥接汽水数据。平台不控制汽水的发布过程。

1.3 用户组与注册机制
系统不开放公开注册。管理员创建用户组，系统自动生成专属邀请码及注册链接。创作者凭邀请码注册后自动归属对应用户组。
注册时填写：邀请码、手机号、密码，并同意用户协议。注册成功后，创作者须在个人中心单独完成实名认证（提交真实姓名、身份证号），通过审核后方可触发发行与打款流程。

1.5 前端三端架构说明
本平台前端由三个独立 Web 应用构成，分别对应三类角色，部署在不同 URL 路径下：
创作者端（Creator App）：供创作者使用，支持邀请码注册入口。
评审端（Reviewer App）：仅供评审人员使用，无注册入口，账号由管理员在管理端创建后下发。
管理端（Admin App）：供平台管理员及超级管理员使用，无注册入口，账号由超级管理员创建。
三端均为独立部署的单页应用（SPA），共享同一套后端 API，但登录入口、权限体系和路由结构互相独立。评审端与管理端不对外公开注册链接。

Phase 1：创作（平台外部）
创作者使用汽水创作实验室或Suno等AI工具在平台外创作歌曲
歌曲自动发布到汽水音乐，产生汽水歌曲ID，在汽水/抖音产生播放量和收益
此阶段完全在本平台外部发生，本平台不介入、不控制汽水的发布过程。
Phase 2：作品入库（创作者端）
自由上传（3步骤）
Step 1 文件上传：拖拽或点击上传音频（WAV/MP3，≤50MB），完成后显示波形预览；可选上传封面图（JPG/PNG/WebP，≤5MB，建议 1:1 比例）
Step 2 元数据填写：填写歌曲信息+AI声明+原创性声明（必须勾选）；额外字段：创作贡献（主导/参与，必填）+ 创作描述（选填，描述创作思路）
Step 3 预览确认：展示完整元数据+音频信息，确认后提交
系统自动生成作品编号（格式：AIMU-YYYY-NNNNNN，如 AIMU-2026-000001）
作品进入 pending_review 状态，进入评审队列
作业提交
创作者在「作业提交」页面查看所在用户组的作业列表
点击进入具体作业 → 上传音频 → 填写动态表单（字段由管理员按组配置）
提交成功后系统同步将作品入库（source=assignment），状态 pending_review，进入评审队列
一个用户对同一作业只允许提交一次；需修改时，在作业结果页重新提交覆盖旧记录（version++，旧评审记录保留可追溯）
Phase 3：作品评审（评审端）
评审在工作台查看待评审数量，进入待评审列表（支持按流派/创作者筛选）
点击「开始评审」进入评审页面：试听（A-B循环、变速）+ 波形标记 + AI预分析报告 + 元数据 + 歌词
填写评分表单：技术熟练度(30%) + 创意立意(40%) + 商业传播潜力(30%)，三维0-100滑块，系统自动计算加权总分
选择快捷评语标签（可叠加），填写专业评语（≥20字必填）
选择发行建议：强烈推荐发行 / 建议修改后发行 / 暂不推荐
提交评审，系统根据推荐等级+分数自动流转状态（详见第3章状态机）
Phase 4：作品发行（管理员端）
「歌曲库管理」按 ready_to_publish 状态筛选待发行作品
发行前系统校验三项条件（全满足方可点击确认发行）：
① 创作者已签署代理发行协议
② 创作者实名认证通过（verified）
③ 作品已绑定ISRC码
校验通过 → 管理员点击「确认发行」→ 状态变更为 published
「作品库批量下载」：按状态/流派/AI工具/评分筛选，勾选批量打包下载（音频+元数据JSON+版权凭证PDF）
「ISRC管理」：查看待申报列表 → 提交申报 → 上传回填文件批量更新ISRC
「发行渠道管理」：歌曲×平台矩阵，记录各平台上架状态
Phase 5：发行状态确认（双轨）

Phase 6：收益导入（管理员端）
从汽水音乐后台导出CSV报表，上传到本平台
系统自动处理：解析 → 去重 → 查映射表 → 名称匹配 → 生成结算预览
管理员处理存疑/未匹配条目（确认/修正/标记无关）
确认导入 → 生成 settlement 记录 → 管理员触发导出/打款
打款时自动检查创作者实名状态，未 verified 则阻断并提示
Phase 7：创作者查收益（创作者端）
「我的收益」展示两个 Tab：平台分发收益 + 汽水音乐收益
平台分发收益：来自 settlements 表，展示歌曲/平台/原始收益/我的分成/结算状态
汽水音乐收益：来自 qishui_details 中 matched_user_id=当前用户的记录，展示按月收益明细
创作者不可见汽水歌曲ID、映射逻辑；收益与评审状态无强依赖关系

8. 边界场景与异常处理
场景
处理规则
先有收益后有提交
导入时名称匹配失败 → pending。创作者后续提交 → 管理员手动关联 → 回溯生成历史结算
汽水歌名与平台歌名不一致
名称匹配失败 → pending。管理员手动匹配。映射保存后下次导入自动命中
CSV包含不属于本平台的歌曲
标记 irrelevant → 后续导入自动跳过，不再出现在待匹配队列
一首歌多月收益
不同 period 同一 qishui_song_id 不算重复，映射表只需一条，多条 revenue_rows 均生成结算
同名歌曲不同创作者
精确匹配多条 → suspect，不自动选择，等管理员确认正确归属
创作者未实名
收益可匹配到账号，但 settle_status 不可推进到 paid，系统前置校验阻断
创作者账号被禁用
结算正常生成，但状态停在 confirmed/exported，管理员人工处理后打款
跨批次文件重复行
全局 UNIQUE(qishui_song_id, period)，无论哪个批次先导入，重复行均标 duplicate 跳过
评审端提交后歌曲状态异常
总分≥80且强烈推荐 → ready_to_publish；建议修改 → needs_revision；暂不推荐 → reviewed。任意情况均生成 reviews 记录
同一作业用户重复提交
UNIQUE(assignment_id, user_id) 约束，重新提交为更新操作（version++），非新增
发行时ISRC未绑定
系统前置校验阻断，toast提示「请先绑定ISRC码」，不生成发行记录
结算金额为0的行
仍生成 settlement 记录（便于历史追溯），settle_status=confirmed，不触发打款流程
映射表解除后重新关联
软删除旧映射，重新建立新映射；旧 settlement 标记 exception，新 settlement 重新生成

9.2 数据隔离原则
创作者只可读取自身关联的 platform_songs、assignment_submissions、settlements 数据
评审只可读取分配给自己（或全平台）的待评审队列，不可修改作品元数据
平台管理员只可操作其角色权限树中 value=true 的功能模块
操作日志对全平台管理员只读，不可删改
archived（已归档）作品对创作者端仍可见（作品库"已入库"Tab 显示），但不出现在作品广场；收益结算不受归档影响；作品卡片上状态角标仍显示「已归档」，以与 reviewed 状态区分，不会被覆盖显示为「已入库」
汽水歌曲ID等映射字段对创作者端完全不可见

10.3 分页规范

10.5 文件存储规范
音频文件统一存储至对象存储（OSS/S3），平台仅存储URL
上传前在前端校验：格式（WAV/MP3）+ 大小（≤50MB）+ 时长（≤10分钟），减少无效流量
音频文件访问需鉴权（签名URL，有效期1小时），防止盗链
头像图片限制：≤2MB，JPEG/PNG/WebP，自动压缩至500px×500px
批量下载ZIP：服务端生成后存储至OSS，URL有效期24小时，24小时后自动清理
10.6 API规范
采用 RESTful 设计；管理端 API 前缀 /api/admin/v1，用户端 /api/v1
所有接口返回统一结构：{ code, message, data, pagination }
认证：JWT Token，Header: Authorization: Bearer <token>
权限中间件：管理端接口统一校验 admin_users.role → admin_roles.permissions[action_key] = true
接口响应时间SLA：普通查询 < 200ms；复杂统计 < 1s；CSV导入异步任务完成 < 30s（1000行内）
限流：同一IP每分钟最多60次API调用；登录接口每IP每分钟最多10次
附录 A：作品编号生成规则
格式：AIMU-{YYYY}-{NNNNNN}
AIMU：平台固定前缀（AI Music Union）
YYYY：四位年份（如2026）
NNNNNN：6位零填充序号，全平台全年唯一自增（如000001, 000002...999999）
示例：AIMU-2026-000001
自增规则：每年1月1日序号从000001重新开始（UNIQUE约束为 UNIQUE(YYYY, NNNNNN)）。
附录 B：ISRC绑定流程
管理员在「ISRC管理」查看待申报队列（status=ready_to_publish 且 isrc IS NULL）
勾选作品，点击「提交申报」→ 生成申报单（含作品编号、表演者、词曲作者等信息）
向ISRC申报机构提交申报（平台外部流程）
收到ISRC码后，在平台上传回填Excel（格式：copyright_code → isrc 两列）
系统批量更新 platform_songs.isrc，完成绑定
绑定完成后，对应作品发行前校验自动通过ISRC项
