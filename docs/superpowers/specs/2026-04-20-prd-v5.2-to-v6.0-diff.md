## 1. 新增章节（Add）

### § 1.4.5 权限粒度实施规范（新增）

**关联 GAP**: GAP-ADMIN-004 / GAP-ADMIN-005 / GAP-ADMIN-062 / GAP-COMM-001 / GAP-COMM-003

#### 1.4.5.1 六种 action 统一词汇表

| action | 含义 | 典型 HTTP 映射 |
|---|---|---|
| `view` | 查看列表/详情 | GET |
| `edit` | 修改单个资源字段（非状态流转） | PUT/PATCH |
| `manage` | 增删资源（状态流转含 DELETE、POST 创建） | POST/DELETE |
| `operate` | 触发业务操作（如触发对账、一键发行） | POST 带 action 参数 |
| `export` | 导出 CSV/Excel | GET `/export` 路径后缀 |
| `settle` | 收益结算类敏感操作（打款、出账期、调整规则） | POST `/revenue/**/pay`、PUT `/revenue/**/settle-status` |

**实施原则**：
- 所有受保护的 API **必须显式声明权限 key**：`requirePermission(request, 'admin.{module}.{action}')`
- 不再依赖 `inferPermissionKey` 按 HTTP 方法自动推断（v5.2 的推断只覆盖 3 种 action，与 6 种设计不一致）
- 侧边栏 permKey 与权限树 key 必须**完全一致**（统一用 underscore 如 `admin.publish_confirm`）

#### 1.4.5.2 group_admin 生效约束

`admin_level = group_admin` 的管理员，访问带 `groupId` 维度的 API 时：
- 创建/编辑/查看本组成员：放行
- 访问非本组或跨组数据：返回 403
- 实施位置：`requirePermission` 同时校验 `groupId ∈ user.managedGroupIds`

#### 1.4.5.3 超级管理员唯一性

建立内置角色 `roleId=1` 时，后端禁止给第二个 `admin_user.roleId=1` 入库：
- POST `/admin/admins` 前判断 `count(roleId=1) >= 1` 则返回 409
- UI 给出"超级管理员已存在"提示

### § 4.17 新增表 token_revocations（吊销黑名单）

**关联 GAP**: GAP-LOOP-010 / GAP-SCHM-014

```sql
CREATE TABLE token_revocations (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  jti VARCHAR(64) NOT NULL UNIQUE COMMENT 'JWT payload.jti',
  user_id INT NOT NULL,
  portal ENUM('creator','reviewer','admin') NOT NULL,
  revoked_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  expires_at DATETIME NOT NULL COMMENT '原 JWT exp，用于定期清理',
  reason VARCHAR(30) COMMENT 'logout/password_change/account_disable',
  INDEX idx_expires (expires_at),
  INDEX idx_user_portal (user_id, portal)
) ENGINE=InnoDB;
```

**用途**：
- `logout` 写入当前 token jti
- `password_change` 批量吊销该用户所有未过期 token
- `account_disable`（禁用账号）同上
- middleware 校验 JWT 时 `SELECT 1 FROM token_revocations WHERE jti = ? LIMIT 1`（配合 cache）
- 定期 cron 清理 `expires_at < NOW() - 1 day`

### § 10.7 文件上传安全规范（新增）

**关联 GAP**: GAP-LOOP-002 / GAP-LOOP-006 / GAP-LOOP-007 / GAP-CRTR-006 ~ 008 / GAP-CRTR-047

所有文件上传路径的安全要求：

1. **上传 Token**
   - 后端签发 HMAC-SHA256 token，payload: `{ userId, keyPrefix, fileType, maxSize, expiresAt, nonce }`
   - 有效期 ≤5 分钟（`storage.uploadTokenTtlSec`）
   - OSS 模式走 `ali-oss signatureUrl(method: 'PUT', expires: 300)`
   - 本地模式 PUT 端点校验 HMAC + 时间戳窗口
   
2. **Magic Bytes 校验**
   - 扩展名校验 + MIME 魔术字节二次校验（本地 PUT 端点读前 16 字节）
   - 支持类型：MP3 (`ID3` / `FF FB`)、WAV (`RIFF ... WAVE`)、PNG (`89 50 4E 47`)、JPEG (`FF D8 FF`)、WebP (`RIFF ... WEBP`)
   - 检测不通过返回 415 + 记录 operation_logs
   
3. **文件访问签名**
   - 所有 audio/cover URL 返回前签名（`signatureUrl(key, { expires: signedUrlTtlSec })`）
   - 本地开发用 `/api/files/[...path]` 中间路由：校验登录 + 签名时间戳
   - 未登录 + 未签名 → 403
   
4. **OSS Bucket 策略**
   - 生产 OSS bucket 必须 **私有读写**，禁止公开读
   - 签名 URL 是唯一访问通道
   
5. **文件 GC**
   - 本地 `public/uploads/*` 下未被 `platform_songs.audioUrl/coverUrl` 引用的文件，每周脚本清理

### § 10.8 API 限流分级（新增）

**关联 GAP**: GAP-LOOP-008 / GAP-LOOP-009 / GAP-CRTR-009 / GAP-CRTR-059 / GAP-ADMIN-093

| 路径前缀 | 限流 | 维度 | 说明 |
|---|---|---|---|
| `POST /api/auth/login` | 10/分钟 | IP | v5.2 已有 |
| `POST /api/auth/sms/send` | 5/分钟 | IP + phone | 防刷短信费 |
| `POST /api/auth/sms/verify` | 10/分钟 | IP + phone | 防爆破 |
| `POST /api/auth/reset-password` | 5/分钟 | IP | 防爆破 |
| `POST /api/upload/token` | 30/分钟 | userId | 防重放 |
| `GET /api/songs/published`、`/api/content` | 60/分钟 | IP | 公开接口防爬 |
| 所有 `/api/*`（tail catch） | 120/分钟 | IP | 全局保护 |
| `POST/PUT/DELETE /api/admin/**` | 100/分钟 | userId | admin 写操作 |

**实施**：
- `middleware.ts` 按路径分级调用 `ipRateLimit(ip, key, limit, window)`
- 多实例部署时，rate-limit 存储从进程内存迁到 **Redis**（`cache.ts` 同理）
- 超限返回 429 + `Retry-After` header

### § 1.2 角色体系

**关联 GAP**: GAP-ADMIN-032

**原文（v5.2）**：
> 评审 查看待评审队列 / 打分评语 / 通过或退回；不可修改元数据

**新文（v6.0）增补**：

> 评审 查看待评审队列 / 打分评语 / 通过或退回；**不可修改作品元数据**。
> 
> 平台管理员 **可修改作品元数据的非核心字段**，用于运营纠错：
> - 允许修改：`genre / bpm / lyrics / lyricist / composer / performer / albumName / albumArtist / creationDesc / aiTools`
> - **禁止修改**：`copyrightCode`（作品编号）、`isrc`、`userId`（归属创作者）、`source`、`assignmentId`
