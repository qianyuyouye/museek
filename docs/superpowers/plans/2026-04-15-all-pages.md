# Museek 全平台页面实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成 Museek 三端所有剩余页面（管理端13页 + 创作者端9页 + 评审端3页 = 25页）

**Architecture:** 所有页面为纯前端 Mock 实现，功能逻辑完全复刻 HTML 原型，视觉严格匹配 PNG 截图。复用已有共享组件（PageHeader, DataTable, StatCard, StatusBadge, AdminModal, AdminTab, SearchBar, PermissionTree）。

**Tech Stack:** Next.js 15 (App Router) + React 19 + TypeScript + Tailwind CSS + shadcn/ui

**核心开发原则：**
- **功能逻辑** → 完全复刻 `docs/管理端.html`、`docs/创作者端.html`、`docs/评审端.html`
- **视觉效果** → 严格匹配 `docs/AI音乐平台/*.png` 截图
- 每个 subagent 必须先用 Read 工具读取对应 PNG 截图，再写代码

### Batch 1：管理端剩余页面（13页，5轮并行）

#### Round 1（4 个并行 agent）

- UI: 读取 `docs/AI音乐平台/内容管理.png`

**Task 2: 作业管理**
- Create: `src/app/(admin)/admin/assignments/page.tsx`
- Mock 数据: 在 `src/lib/mock/admin.ts` 添加 `MOCK_ASSIGNMENTS`, `MOCK_ASSIGNMENT_SUBMISSIONS`, `MOCK_ASSIGNMENT_FIELDS`

  - 表单字段配置弹窗：9个字段的必填/默认值设置，支持变量 {realName}/{songTitle}
  - 作业详情子页：3个统计卡片 + 提交列表表格 + 底部作业信息卡片
- UI: 参考管理端通用表格风格

**Task 3: 合同台账**

**Task 4: 评审绩效**
