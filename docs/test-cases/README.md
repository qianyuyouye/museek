## 用例编号规则

格式：`TC-{端}-{模块序号}-{用例序号}`

| 前缀 | 含义 |
|------|------|
| `TC-AUTH-NNN` | 认证相关 |
| `TC-C-MM-NNN` | 创作者端（MM 对应小节 01-10） |
| `TC-R-MM-NNN` | 评审端 |
| `TC-A-MM-NNN` | 管理端 |
| `TC-E2E-NNN` | 端到端闭环 |
| `TC-BD-NNN` | 边界/异常 |

示例：`TC-C-02-005` = 创作者端 - 作品上传（7.1.2）- 第 5 条用例。

## 优先级

| 级别 | 含义 | 典型场景 |
|------|------|---------|
| P0 | 阻塞级，核心主流程 | 登录、作品提交、评审、发行、结算 |
| P1 | 重要功能 | 筛选、搜索、导出、权限校验 |
| P2 | 边界 / 体验增强 | UI 提示、排序、图表、可选字段 |

## 通用前置

除非特别标注，以下条件默认成立：

- 后端服务、MySQL、对象存储（OSS / 本地）均正常
- 浏览器为 Chrome / Edge 最新版
- 对应角色已登录，持有相应 `{portal}.{menu}.{action}` 权限 Key
- 测试数据：至少 1 个有效邀请码 `TESTCODE8`、1 个创作者、1 个评审、超管 `admin/Abc12345`、1 首作品、1 个用户组

## 通用约定

- 表单校验：`onBlur` 即时反馈 + 提交时汇总校验
- 管理端所有写操作 100% 写入 `operation_logs`（P2-2 已覆盖）
- 列表默认 `page_size=20`，最大 100（PRD 10.3）
- 所有接口返回统一 `{ code, message, data, pagination }`
- 作品编号格式固定 `AIMU-YYYY-NNNNNN`，全年唯一自增
- 状态枚举严格对照 PRD 附录 D，不得新增或变造

## 维护说明

- 新增页面 / 按钮：先在对应模块文件追加用例，再更新本文件索引计数
- PRD 改版：以 `docs/prd_text.txt` 为唯一基线，用例 ID 不复用
- 执行记录：建议在 `docs/test-cases/runs/` 下以日期命名记录执行结果
