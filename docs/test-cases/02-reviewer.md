# 评审端测试用例

对应 PRD §7.2。覆盖 5 页：工作台、待评审列表、评审页面、我的绩效、个人中心。
核心规则对照 PRD §3 状态机、§7.2.3 评分公式。

## 通用前置

- 登录评审账号 `reviewer01 / Abc12345`
- 具备权限：`teacher.*` 全部查看/操作
- 测试数据：至少 3 首 `pending_review` 作品（不同流派、来源）

---

## 01. 工作台 · PRD 7.2.1

| 用例 ID | 按钮 / 功能点 | 前置 / 步骤 | 预期结果 | 优先级 |
|--------|--------------|-------------|----------|--------|
| TC-R-01-001 | "待评审数"卡片 | 访问 `/review` | 数字 = `COUNT(*) FROM platform_songs WHERE status=pending_review` | P0 |
| TC-R-01-002 | 卡片实时更新 | 评审提交 1 条后刷新 | 数字 -1 | P0 |
| TC-R-01-003 | "本月已完成"卡片 | 访问 | 数字 = `COUNT(*) FROM reviews WHERE reviewer_id=me AND MONTH(reviewed_at)=当前月` | P1 |
| TC-R-01-004 | "平均用时"卡片 | 访问 | `AVG(duration_seconds)` 本月 | P1 |
| TC-R-01-005 | "平均评分"卡片 | 访问 | `AVG(total_score)` 本月 | P1 |
| TC-R-01-006 | 评审趋势图 | 查看 | 近 30 天每日评审量柱状图 | P1 |

| TC-R-01-008 | "进入评审列表"按钮 | 点击 | 跳转 `/review/queue` | P0 |
